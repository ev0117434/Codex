- Демо рабочего pipeline (минимум 1 символ на 4 источниках).
- Отчет по SLA чернового уровня: freshness и snapshot latency.
- Список блокеров и внешних зависимостей.


## Принятые решения от заказчика
- Только `spot vs perpetual`, без quarterly.
- Символы не исключаются, фильтр ликвидности не применяется.
- Snapshot только в файл.
- Целевой latency: p95/p99 = 500ms.
- Допустимая stale-доля: 50%.
- Доступность: 100%.
- RTO/RPO не используются.
- Proxy/fallback не используются.
- Истинное время staleness: `ts_receive`.
- Деплой: bare metal VPS.
