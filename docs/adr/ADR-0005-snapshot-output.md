# ADR-0005: Snapshot as primary output artifact

- Status: Accepted
- Date: 2026-03-09

## Context
На v1.1 нужен простой и проверяемый способ публикации результата без тяжёлой инфраструктуры.

## Decision
Публиковать периодический snapshot в текстовый файл атомарной записью (temp + rename).

## Rationale
- Простая интеграция с внешними процессами.
- Лёгкая диагностика человеком.
- Надёжная семантика публикации при атомарном rename.

## Consequences
- Плюсы: минимальный ops-cost, прозрачность.
- Минусы: нет встроенной историчности и query API.
